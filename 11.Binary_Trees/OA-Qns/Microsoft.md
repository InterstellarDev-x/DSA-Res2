# Microsoft — Binary Trees OA Questions

Microsoft assessments emphasize solid command of the classic, foundational tree operations — symmetry checks, inversions, level-order processing, and structural comparisons. Problems are typically Easy-to-Medium and test whether you can write clean, bug-free recursion and BFS quickly. Readable code and correct handling of null children matter as much as the algorithm.

## OA Tips

- Microsoft OAs frequently use the Codility/HackerRank platform; read the I/O format carefully since trees may be given as arrays or edge lists rather than a prebuilt structure.
- Easy problems still have hidden edge cases — empty tree, single node, and asymmetric/null subtrees — handle them before submitting.
- Both recursive and iterative (BFS/DFS with explicit stack/queue) solutions are accepted; pick the one you can write bug-free fastest.
- Partial scoring is common, so submit a working brute-force first, then optimize if time allows.
