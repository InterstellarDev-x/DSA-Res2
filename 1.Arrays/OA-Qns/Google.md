## OA Overview

Google's hiring process typically does not have a traditional OA — instead it uses phone screens and onsite rounds on Google Docs / shared coding environments. However, Google's Kick Start, Code Jam, and internal screening rounds feature array problems heavily. The questions below are from Google phone screens and L3/L4 interviews.

## Common Patterns in Google OAs

| Pattern | Frequency |
|---------|-----------|
| [Kadane's Algorithm](../Patterns/Kadane's%20Algorithm.md) | Very High |
| [Prefix Sum](../Patterns/Prefix%20Sum.md) | High |

| In-place Matrix Operations | Medium |
| [Cyclic Sort](../Patterns/Cyclic%20Sort.md) | Low–Medium |

> **Note:** Google favors questions that test mathematical insight and edge case handling. Expect follow-ups asking to optimize from O(n²) to O(n) or O(1) space.

## Notes

- Google values **clean, readable code** over clever one-liners
- Always discuss time and space complexity proactively
- Prepare for: "Can you do it in O(1) extra space?"
